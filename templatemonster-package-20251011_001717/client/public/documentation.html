1. Installation

Prerequisites

Before installing this template, make sure you have the following installed:

  • Node.js (version 16 or higher)
  • npm or yarn package manager
  • Git (for version control)

Quick Start

  1. Extract the ZIP file to your desired directory
  2. Open terminal/command prompt in the project directory
  3. Install dependencies:
    npm install
  4. Start the development server:
    npm start
  5. Open your browser and navigate to http://localhost:3000
Success! Your automation dashboard should now be running locally. You'll see the main dashboard with all automation tools and features.

2. Configuration

Environment Setup

Create a .env file in the root directory with the following variables:

REACT_APP_API_URL=http://localhost:5000 REACT_APP_STRIPE_PUBLISHABLE_KEY=your_stripe_key REACT_APP_GOOGLE_ANALYTICS_ID=your_ga_id

Database Configuration

This template uses MongoDB for data storage. Configure your database connection:

  1. Install MongoDB locally or use MongoDB Atlas (cloud)
  2. Update the database connection string in server.js
  3. Run database migrations if needed

API Configuration

Configure your backend API settings in client/src/config/api.js:

const API_BASE_URL = process.env.REACT_APP_API_URL || 'http://localhost:5000'; const API_ENDPOINTS = { auth: '/api/auth', leads: '/api/leads', campaigns: '/api/campaigns' };

3. Customization

Branding & Colors

Customize your brand colors and styling:

  1. Open client/src/theme/colors.js
  2. Update the color palette:
    export const colors = { primary: '#667eea', secondary: '#764ba2', accent: '#f093fb', // Add your custom colors };
  3. Save and refresh your application

Logo & Images

Replace default images with your own:

  • Logo: Replace client/public/logo.svg
  • Favicon: Replace client/public/favicon.ico
  • Hero Images: Update images in client/src/assets/

Content Customization

Update text content throughout the application:

  • Homepage: Edit client/src/pages/Home.js
  • About Page: Edit client/src/pages/About.js
  • Contact Info: Update client/src/config/business.js

Contact Form Setup

Configure the contact form to send emails:

  1. Open client/src/components/ContactForm.js
  2. Update the form action URL
  3. Configure your email service (SendGrid, Mailgun, etc.)
  4. Test the form functionality
Tip: Use environment variables for sensitive information like API keys and email credentials.

5. Deployment

Production Build

Create a production build of your application:

npm run build

Deployment Options

This template can be deployed to various platforms:

Vercel (Recommended)

  1. Connect your GitHub repository to Vercel
  2. Set environment variables in Vercel dashboard
  3. Deploy automatically on every push

Netlify

  1. Drag and drop the build folder to Netlify
  2. Configure environment variables
  3. Set up custom domain if needed

Traditional Hosting

  1. Upload the build folder contents to your web server
  2. Configure your web server for single-page application routing
  3. Set up SSL certificate for HTTPS
Important: Always test your production build locally before deploying to ensure everything works correctly.

6. Support & Troubleshooting

Common Issues

Build Errors

If you encounter build errors:

  1. Clear node_modules: rm -rf node_modules
  2. Clear npm cache: npm cache clean --force
  3. Reinstall dependencies: npm install
  4. Try building again: npm run build

API Connection Issues

If the frontend can't connect to the backend:

  1. Check if the backend server is running
  2. Verify API URL in environment variables
  3. Check CORS settings in backend configuration
  4. Ensure both frontend and backend are using the same protocol (HTTP/HTTPS)

Styling Issues

If styles aren't loading correctly:

  1. Check if CSS files are properly imported
  2. Verify that all image assets are in the correct directories
  3. Clear browser cache and hard refresh
  4. Check for console errors in browser developer tools

Performance Optimization

  • Optimize images before uploading
  • Enable gzip compression on your server
  • Use a CDN for static assets
  • Implement lazy loading for images
  • Minimize third-party scripts

Browser Compatibility

This template supports all modern browsers:

  • Chrome 90+
  • Firefox 88+
  • Safari 14+
  • Edge 90+
